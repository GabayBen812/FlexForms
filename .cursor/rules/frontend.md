# Frontend Guidelines (React + TypeScript)

- Use **React functional components** only (no class components).
- Always use **React Hooks** and **Context** for state management.
- Use **Tailwind CSS** for styling (no inline styles or CSS files).
- Keep components small and focused.
- Use `camelCase` for file names and function names.
- Use TypeScript interfaces and props validation.
- Keep folder structure consistent (`components/`, `pages/`, `hooks/`, etc.).
